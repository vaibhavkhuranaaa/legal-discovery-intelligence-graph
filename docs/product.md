# Product Specification — Legal Discovery Intelligence Graph

## User

A litigation support / eDiscovery analyst (or the attorney they support) who has been handed a
document production and must answer investigative questions quickly: who was involved, what
happened, when, and where is the evidence.

Secondary audience: technical interviewers evaluating the author's AI engineering ability — the
product must be demonstrable end-to-end in a public deployment within ~90 seconds.

## Problem

Document review is dominated by keyword search over unstructured text. Keyword search misses
paraphrase ("payment" vs "wire transfer"), and flat search results hide the relationships that
matter in an investigation: who communicated with whom, which entities co-occur across
documents, and how events order in time. Analysts manually rebuild this context in spreadsheets.

## What the Product Does

Over a corpus of discovery documents, the application provides:

1. **Ask a question, get cited evidence** — semantic retrieval returns relevant document chunks
   with document-level citations, not generated claims without provenance.
2. **Expand into the graph** — from any retrieved evidence, see the connected entities
   (people, organizations, amounts, projects) and the other documents/events they touch.
3. **Timeline view** — dated events extracted from the corpus, filterable by entity, to
   reconstruct chronology.
4. **Evaluation transparency** — extraction and retrieval quality reported as
   precision/recall/F1 against gold labels, so quality claims are reproducible.

## Core Workflows

- **Investigative question** → vector search (pgvector) → graph expansion (Neo4j) → answer view
  with cited chunks + entity subgraph.
- **Entity-centric browse** → pick an entity → all mentions, related entities, documents, and
  events.
- **Timeline reconstruction** → filter events by entity or date range → Plotly timeline.
- **Evaluation review** → metrics page showing the current scored run against gold labels.

## Scope

- Synthetic discovery corpus (emails, contracts, memos, invoices, meeting notes) generated by
  `scripts/bootstrap_data.py`, embedded around a coherent fictional investigation scenario.
- Entity types: person, organization, money, date, project, location.
- English-language documents only.
- Public deployment on Streamlit Community Cloud with Supabase and Neo4j AuraDB free tiers.

## Non-Goals

- Not a legal-advice or privilege-review tool; no responsiveness/privilege classification.
- No real client, confidential, or personal data — ever.
- No OCR / scanned-document ingestion (documents are generated as clean text).
- No multi-tenant auth or role-based access control (single public demo corpus).
- No LLM-generated free-form answers without retrieval grounding; evidence is always cited.

## Synthetic-Data Policy

All corpus content is fictional and generated deterministically by seedable code committed to
this repository. Names, companies, amounts, and events must not correspond to real parties. This
policy is enforced in `CLAUDE.md`/`AGENTS.md` and makes the corpus fully redistributable,
reproducible, and safe for a public deployment — and enables exact gold labels, since ground
truth is known at generation time (see `DATA_AND_EVALUATION.md`).
